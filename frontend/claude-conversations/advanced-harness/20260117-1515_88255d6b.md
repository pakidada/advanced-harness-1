# Jewelry Image Generator

Generate visually consistent jewelry product images with a lovely, princess-like pink aesthetic using Replicate MCP and the Google Nano Banana model.

## Purpose

Transform jewelry product concept photos into styled images featuring:
- Lovely, princess-like aesthetic
- Pink and rose gold color palette
- Elegant jewelry box presentation
- **Paired reference images â†’ Single output per pair**

---

## File Naming Convention (IMPORTANT)

### Pairing System

Files are paired by naming pattern: `{number}-{sub}`

**Examples:**
```
1-1.jpg, 1-2.jpg  â†’ Pair 1 â†’ Output: result_1.jpg
2-1.jpg, 2-2.jpg  â†’ Pair 2 â†’ Output: result_2.jpg
3-1.jpg, 3-2.jpg  â†’ Pair 3 â†’ Output: result_3.jpg
```

### How Pairing Works

| Input Files | Pair # | Output |
|-------------|--------|--------|
| `1-1.jpg` + `1-2.jpg` | 1 | 1 image |
| `2-1.jpg` + `2-2.jpg` | 2 | 1 image |
| `3-1.jpg` + `3-2.jpg` | 3 | 1 image |

**Rule:** Same prefix number = same pair â†’ combined as references â†’ 1 output

---

## Workflow

### Step-by-Step Process

1. **Identify Pairs from Filenames**
   ```
   Input: 1-1.jpg, 1-2.jpg, 2-1.jpg, 2-2.jpg

   Pairs:
   - Pair 1: [1-1.jpg, 1-2.jpg]
   - Pair 2: [2-1.jpg, 2-2.jpg]
   ```

2. **Compress & Upload Each Image**
   ```bash
   # Compress
   sips -Z 512 -s format jpeg -s formatOptions 50 input.jpg --out /tmp/compressed.jpg

   # Upload to litterbox (1 hour expiry)
   curl -sF "reqtype=fileupload" -F "time=1h" \
        -F "fileToUpload=@/tmp/compressed.jpg" \
        https://litterbox.catbox.moe/resources/internals/api.php
   ```

3. **Generate One Image Per Pair**
   ```yaml
   mcp__replicate__create_predictions:
     version: "google/nano-banana"
     input:
       prompt: "Transform: place this jewelry in pink velvet box..."
       image_input: ["url_1-1", "url_1-2"]  # Both images of pair
       aspect_ratio: "4:5"
       output_format: "jpg"
   ```

4. **Save Output**
   ```bash
   curl -o result_1.jpg "[output_url]"
   ```
