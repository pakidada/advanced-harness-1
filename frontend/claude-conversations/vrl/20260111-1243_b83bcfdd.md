# VRL PPT Brand Guidelines

Minimal, modern brand guidelines for VRL presentation materials.

## When to Use

- Creating PowerPoint/Keynote/Google Slides presentations
- Designing slide decks for pitches, reports, or meetings
- Any visual content requiring VRL brand consistency

## Brand Colors

### Primary Palette

| Color | Hex | RGB | Usage |
|-------|-----|-----|-------|
| **Lime** | `#BDFF00` | 189, 255, 0 | Primary accent, highlights, CTAs |
| **Dark** | `#0F172A` | 15, 23, 42 | Text, headers, dark backgrounds |
| **White** | `#FFFFFF` | 255, 255, 255 | Backgrounds, text on dark |

### Accent Colors

| Color | Hex | RGB | Usage |
|-------|-----|-----|-------|
| **Orange** | `#FF6B35` | 255, 107, 53 | Secondary accent, emphasis |
| **Lime Dark** | `#84CC16` | 132, 204, 22 | Supporting accent |
| **Slate** | `#1E293B` | 30, 41, 59 | Cards, sections |

### Background Tints

| Color | Hex | Usage |
|-------|-----|-------|
| Light Gray | `#F8FAFC` | Subtle backgrounds |
| Slate 600 | `#475569` | Muted text |
| Slate 300 | `#CBD5E1` | Tertiary text, borders |

## Logo

### Files
- `assets/logo.png` - Primary Q logo (use this)

### Usage Rules
- **Minimum size**: 24px height
- **Clear space**: Equal to logo height on all sides
- **Colors**: Black on light backgrounds, white on dark backgrounds
- Never distort, rotate, or add effects

### Placement
- Top-left corner of title slides
- Bottom-right corner of content slides
- Centered for closing slides

## Typography

### Recommended Fonts
- **Headlines**: Inter Bold, SF Pro Bold, or system sans-serif bold
- **Body**: Inter Regular, SF Pro Regular, or system sans-serif
- **Code**: SF Mono, Fira Code, or system monospace

### Hierarchy
- Title: 44-60px, Bold
- Subtitle: 24-32px, SemiBold
- Body: 18-24px, Regular
- Caption: 14-16px, Regular

## Slide Design Principles

### 1. Minimal Layout
- One idea per slide
- Maximum 6 bullet points
- Generous white space (40%+ of slide)

### 2. Color Application
- Dark text (`#0F172A`) on light backgrounds
- Lime (`#BDFF00`) for key highlights only
- Orange (`#FF6B35`) for secondary emphasis

### 3. Visual Hierarchy
```
[Logo - small, top-left]

HEADLINE
Subtitle or supporting text

â€¢ Key point one
â€¢ Key point two
â€¢ Key point three

[Optional: Single visual/chart]
```

### 4. Contrast Guidelines
- High contrast for text (dark on light)
- Lime accent on dark backgrounds
- Never lime text on white (poor contrast)

## Quick Reference

### DO
- Use exact hex values
- Keep slides clean and minimal
- One accent color per slide
- Generous margins (10%+ on all sides)

### DON'T
- Use gradients on logo
- Mix too many accent colors
- Overcrowd slides with text
- Use lime for body text

## Slide Templates

### Title Slide
- Background: `#0F172A` (dark) or `#FFFFFF` (light)
- Logo: Top-left, white on dark / black on light
- Title: Center, large (60px), Bold
- Subtitle: Below title, smaller (24px)

### Content Slide
- Background: `#FFFFFF`
- Logo: Bottom-right, small (24px)
- Header: Top-left, `#0F172A`, 32px Bold
- Body: `#0F172A`, 20px Regular
- Accent: Lime highlights sparingly

### Section Divider
- Background: `#BDFF00` or `#0F172A`
- Text: Centered, contrasting color
- Logo: Bottom-right

### Closing Slide
- Background: `#0F172A`
- Logo: Center, large
- Contact/CTA: Below logo, white text

# PPTX creation, editing, and analysis

## Overview

A user may ask you to create, edit, or analyze the contents of a .pptx file. A .pptx file is essentially a ZIP archive containing XML files and other resources that you can read or edit. You have different tools and workflows available for different tasks.

## Reading and analyzing content

### Text extraction
If you just need to read the text contents of a presentation, you should convert the document to markdown:

```bash
# Convert document to markdown
python -m markitdown path-to-file.pptx
```

### Raw XML access
You need raw XML access for: comments, speaker notes, slide layouts, animations, design elements, and complex formatting. For any of these features, you'll need to unpack a presentation and read its raw XML contents.

#### Unpacking a file
`python ooxml/scripts/unpack.py <office_file> <output_dir>`

**Note**: The unpack.py script is located at `skills/pptx/ooxml/scripts/unpack.py` relative to the project root. If the script doesn't exist at this path, use `find . -name "unpack.py"` to locate it.

#### Key file structures
* `ppt/presentation.xml` - Main presentation metadata and slide references
* `ppt/slides/slide{N}.xml` - Individual slide contents (slide1.xml, slide2.xml, etc.)
* `ppt/notesSlides/notesSlide{N}.xml` - Speaker notes for each slide
* `ppt/comments/modernComment_*.xml` - Comments for specific slides
* `ppt/slideLayouts/` - Layout templates for slides
* `ppt/slideMasters/` - Master slide templates
* `ppt/theme/` - Theme and styling information
* `ppt/media/` - Images and other media files

#### Typography and color extraction
**When given an example design to emulate**: Always analyze the presentation's typography and colors first using the methods below:
1. **Read theme file**: Check `ppt/theme/theme1.xml` for colors (`<a:clrScheme>`) and fonts (`<a:fontScheme>`)
2. **Sample slide content**: Examine `ppt/slides/slide1.xml` for actual font usage (`<a:rPr>`) and colors
3. **Search for patterns**: Use grep to find color (`<a:solidFill>`, `<a:srgbClr>`) and font references across all XML files

### Layout Tips
**When creating slides with charts or tables:**
- **Two-column layout (PREFERRED)**: Use a header spanning the full width, then two columns below - text/bullets in one column and the featured content in the other. - **Full-slide layout**: Let the featured content (chart/table) take up the entire slide for maximum impact and readability
- **NEVER vertically stack**: Do not place charts/tables below text in a single column - this causes poor readability and layout issues

## Editing an existing PowerPoint presentation

When edit slides in an existing PowerPoint presentation, you need to work with the raw Office Open XML (OOXML) format. This involves unpacking the .pptx file, editing the XML content, and repacking it.

### Workflow
1. **MANDATORY - READ ENTIRE FILE**: Read [`ooxml.md`](ooxml.md) (~500 lines) completely from start to finish.  **NEVER set any range limits when reading this file.**  Read the full file content for detailed guidance on OOXML structure and editing workflows before any presentation editing.

3. Edit the XML files (primarily `ppt/slides/slide{N}.xml` and related files)

5. Pack the final presentation: `python ooxml/scripts/pack.py <input_directory> <office_file>`

## Converting Slides to Images

To visually analyze PowerPoint slides, convert them to images using a two-step process:

1. **Convert PPTX to PDF**:
   ```bash
   soffice --headless --convert-to pdf template.pptx
   ```

2. **Convert PDF pages to JPEG images**:
   ```bash
   pdftoppm -jpeg -r 150 template.pdf slide
   ```
   This creates files like `slide-1.jpg`, `slide-2.jpg`, etc.

Options:
- `-r 150`: Sets resolution to 150 DPI (adjust for quality/size balance)
- `-jpeg`: Output JPEG format (use `-png` for PNG if preferred)
- `-f N`: First page to convert (e.g., `-f 2` starts from page 2)
- `-l N`: Last page to convert (e.g., `-l 5` stops at page 5)
- `slide`: Prefix for output files

Example for specific range:
```bash
pdftoppm -jpeg -r 150 -f 2 -l 5 template.pdf slide  # Converts only pages 2-5
```
