# Evan Insight Blog Writer

evan-insight 블로그용 투자 분석 글 작성 스킬.

## When to Use

- evan-insight 블로그에 투자 분석 글 작성 시
- 기업/주식 분석 콘텐츠 작성 시
- 복잡한 투자 정보를 쉽게 풀어쓸 때
- 어그로성 제목/도입부가 필요할 때
- 섹터/시장 분석 글 작성 시

## 핵심 원칙

### 1. 두괄식 어그로 구조

제목: 자극적이고 클릭을 유도하는 제목
- 긍정: "Next 구글, NVIDIA가 될 회사를 찾았습니다"
- 부정: "Datadog, 지금 사면 안 됩니다"
- 분석: "SaaS 시장이 무너지고 있습니다"

첫 문단: 결론부터 + 핵심 숫자
```markdown
결론부터 말합니다.

SaaS 시장의 신규 매출이 29% 폭락했습니다.

"경기 침체 때문 아냐?"

아닙니다. 더 무서운 일이 벌어지고 있습니다.
```

반박 예상 후 반전:
```markdown
"또 AI 회사?" 하실 수 있어요. 근데 이 회사는 좀 다릅니다.

"근데 Datadog은 다르잖아. AI 수혜주잖아."

아닙니다. Datadog도 SaaS입니다.
```

### 2. 용어 설명 필수 (가장 중요)

모든 전문 용어는 첫 등장 시 반드시 쉽게 풀어서 설명해야 합니다.

"먼저, X가 뭔지부터" 패턴 사용:
```markdown
## 먼저, SaaS가 뭔지부터

SaaS(사스)는 "Software as a Service"의 줄임말입니다.

쉽게 말해서 인터넷으로 빌려 쓰는 소프트웨어예요.

예전에는 CD 사서 컴퓨터에 설치했잖아요. 이제는 그냥 웹사이트 접속해서 월 구독료 내고 씁니다.
```

본문 중 용어 설명 (괄호 활용):
```markdown
P/E(주가수익비율)는 "투자금 회수에 몇 년 걸리나"를 보는 겁니다.

CFO(재무 담당 임원)들이 제일 먼저 쳐다볼 대상이에요.

오픈소스(무료 공개 소프트웨어)를 씁니다. Grafana, Prometheus 같은 거요.
```

전문 용어 → 쉬운 설명 매핑:

| 전문 용어 | 쉬운 표현 |
|----------|----------|
| SaaS | 인터넷으로 빌려 쓰는 소프트웨어. 월 구독료 내고 씀 |
| ARR | 1년 치 매출 기준 |
| IPO | 주식시장 상장 |
| 밸류에이션 | 회사 가치 |
| P/S | 매출 대비 주가가 얼마나 높은지 보는 지표 |
| P/E | 투자금 회수에 몇 년 걸리나를 보는 지표 |
| API | 다른 회사들에게 기능을 빌려주는 것 |
| 흑자 전환 | 돈을 벌기 시작하는 것 |
| 시가총액 | 회사 전체 가치 |
| 토큰 | 글자 |
| GPU/TPU | 특수한 컴퓨터 칩 |
| CFO | 재무 담당 임원 |
| 번들 | 끼워주는 서비스 |
| 모니터링 | 클라우드 서비스의 CCTV 같은 것. 문제 생기면 알려줌 |

금액은 원화로:
- $1B → 약 1조 4천억 원
- $100M → 약 1,400억 원
- $230M → 약 3,300억 원

### 3. 문장 스타일

짧고 임팩트 있게:
```markdown
❌ "이 회사는 매우 빠른 성장률을 보여주고 있으며 시장에서 주목받고 있습니다."
✅ "1년 만에 9배. 역대급입니다."

❌ "이 서비스의 가격이 높다는 점이 문제점으로 지적되고 있습니다."
✅ "문제는, 이 서비스가 비쌉니다."
```

질문으로 독자 참여 유도:
```markdown
"여러분은 지금 어떤 AI를 쓰고 계신가요?"
"왜 이런 격차가 생길까요?"
"이게 왜 중요할까요?"
"무슨 의미일까요?"
```

비유와 은유 활용:
```markdown
"한 바구니에 계란을 담지 않았습니다."
"쉽게 말해서 클라우드 서비스의 CCTV예요."
"금 캐는 사람보다 곡괭이 파는 사람이 이깁니다."
```

### 5. 금지 사항

마크다운 볼드 사용 금지:
- ❌ 절대로 ** 사용하지 마세요. 렌더링이 깨집니다.
- ✅ 대신 ## 헤딩이나 일반 텍스트 사용

AI 티 나는 표현 금지:
- ❌ "매우 인상적입니다"
- ❌ "주목할 만합니다"
- ❌ "흥미롭게도"
- ❌ "살펴보겠습니다"
- ❌ "~해 보도록 하겠습니다"

마케팅 언어 금지:
- ❌ "혁신적인", "획기적인", "놀라운"
- ✅ 대신 구체적 숫자로 보여주기

### 6. 톤 선택 가이드

긍정적 글:
- 기회를 강조
- 리스크 섹션은 짧게
- "왜 이 회사가 유리할까요?" 형태로 프레이밍

부정적 글:
- 위험을 직접적으로 지적
- "사지 마세요", "지금은 아닙니다" 등 명확한 결론
- 반박 예상 → 재반박 구조로 설득력 강화
- "~일까요?" 수사적 질문으로 독자가 스스로 판단하게 유도

### 7. 마무리 패턴

긍정적 마무리:
```markdown
구글 상장 전에 투자한 사람들은 "검색이 뭐가 대단해?"라는 말을 들었습니다.
NVIDIA 살 때도 "게임 칩 회사가 뭐?"라는 소리를 들었습니다.

지금 [회사명]을 보는 시선도 비슷합니다. "또 AI 회사잖아."

근데 [핵심 변화]. 그 중심에 있는 회사입니다.
```

부정적 마무리:
```markdown
[시장 변화]가 재편되고 있습니다.

"[긍정 논리]"라고 불리던 회사도 예외가 아닙니다.

[핵심 위험 요약]

[밸류에이션]짜리 [카테고리]를 지금 살 이유가 없습니다.

[대안 제시]해도 됩니다.

서두를 필요 없습니다.
```
