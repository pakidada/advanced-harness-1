# Threads 글쓰기 스킬

## Purpose

주제를 받으면 11가지 템플릿 중 가장 적합한 형식으로 Threads(스레드) 글을 작성합니다.

## When to Use

- 사용자가 스레드/Threads 글을 써달라고 할 때
- SNS용 짧은 인사이트 글을 요청할 때
- 특정 주제로 공감형/정보형 글을 작성할 때

## 사용법

1. **주제만 던지기**: "AI 자동화로 시간 절약한 이야기" → 가장 적합한 템플릿 자동 선택
2. **템플릿 지정**: "넘버링으로 써줘: 스타트업 대표가 잘한 일" → 지정 템플릿 사용
3. **여러 버전**: "3가지 버전으로 써줘: 번아웃" → 다른 템플릿으로 3개 생성

## 템플릿 목록

| # | 이름 | 적합한 주제 |
|---|------|------------|
| 1 | 증가하는 글자수 | 짧은 인사이트, 감성적 메시지 |
| 2 | 넘버링 글쓰기 | 경험 나열, 리스트형 콘텐츠 |
| 3 | 나만의 원칙 | 대응법, 규칙, 철학 |
| 4 | YES/NO | 상황별 해야할 것/하지 말 것 |
| 5 | 일상 인사이트 | 관찰에서 얻은 깨달음 |
| 6 | 실패 극복기 | 실패→극복→성장 스토리 |
| 7 | 나만의 전략 | 수익화, 방법론 공유 |
| 8 | 효율화 이야기 | 자동화, 프로세스 개선 |
| 9 | 일잘러의 특징 | 인물 관찰, 교훈 |
| 10 | 책 인사이트 | 독서 후기, 핵심 배움 |
| 11 | 시간 관리 | 루틴, 습관, 생산성 |

각 템플릿의 상세 구조와 예시는 [TEMPLATES.md](TEMPLATES.md) 참조.

## 글쓰기 원칙

### 문체
- **구어체**: "~더라고요", "~거든요", "~있어요" 등 자연스러운 말투
- **짧은 문장**: 한 문장당 15~25자 이내 권장
- **줄바꿈 적극 활용**: 가독성을 위해 2~3문장마다 빈 줄
- **이모지 최소화**: 본문에는 쓰지 않음 (제목에만 선택적)

### 톤
- 1인칭 시점 ("저는", "제가")
- 경험 기반 서술 (추상적 조언 X)
- 공감 유도 (독자가 "나도 그래" 할 수 있게)
- 겸손하지만 확신 있는 어조

### 구조
- 첫 줄에서 호기심 유발 (스크롤 멈추게)
- 중간에 구체적 사례/숫자 포함
- 마지막에 여운 남기는 한 줄

### 금지사항
- 해시태그 넣지 않기
- "오늘의 인사이트", "꿀팁" 같은 진부한 표현 금지
- 과도한 감탄사/이모지 금지
- 광고성 문구 금지

## 템플릿 자동 선택 기준

주제 키워드를 분석하여 자동 매칭:

- **실패/극복/성장** → 6. 실패 극복기
- **돈/수익/매출/방법** → 7. 나만의 전략
- **자동화/효율/시간절약** → 8. 효율화 이야기
- **원칙/규칙/대응** → 3. 나만의 원칙
- **해야할/하지말** → 4. YES/NO
- **일잘러/동료/상사** → 9. 일잘러의 특징
- **책/독서/읽고** → 10. 책 인사이트
- **루틴/아침/시간** → 11. 시간 관리
- **관찰/깨달음/문득** → 5. 일상 인사이트
- **목록/리스트/잘한일** → 2. 넘버링 글쓰기
- **그 외 짧은 메시지** → 1. 증가하는 글자수

## 출력 형식

```
[템플릿 이름] 으로 작성

---

(글 본문)

---
```

본문만 깔끔하게 출력. 부가 설명 없이 바로 복사해서 쓸 수 있게.

## 클립보드 복사

글 작성 후 사용자가 "복사", "클립보드", "copy" 등을 요청하면 본문만 `pbcopy`로 클립보드에 복사한다.

```bash
printf '%s' '본문 내용' | pbcopy
```

- 구분선(`---`), 템플릿 이름 등 메타 정보는 제외하고 **본문만** 복사
- `dangerouslyDisableSandbox: true` 필요 (pbcopy는 샌드박스에서 차단됨)
