## 1. 증가하는 글자수

### 구조
```
짧게 시작
열다섯 글자로 써요
스무글자로 늘려서 이야기해볼게요
마지막은 서른자로 완성하면서 인사이트를 남기죠
```

### 핵심 규칙
- 4줄로 구성
- 각 줄의 글자수가 점점 증가 (약 7→15→20→30자)
- 마지막 줄에 인사이트/교훈 담기

### 예시
```
글이 안써질때는
그저 손을 움직여볼 필요가 있어요.
매일 조금이라도 키보드를 두드려야 하죠.
결국 글은 쓰지 않는 것보다 쓰는 게 늘 답이더라고요.
```

## 2. 넘버링 글쓰기

### 구조
```
[특정 타깃/주제/현상]에 대한 생각

1. [내가 진행한 실제 행동 1,2,3...]
2. [내가 생각하는 좋은 점 1,2,3....]
3. [공감이 될만한 내용 1,2,3...]
4. ..
5. ..
```

### 핵심 규칙
- 제목은 구체적 경험 기반
- 넘버링 항목은 5~7개
- 각 항목은 간결하게 한 줄
- 뒤로 갈수록 글자수 약간 증가

### 예시
```
세돌 아이를 키운 후, 지금까지 가장 잘한 일

1. 아이 영상을 많이 남겨둔 것
2. 전국 곳곳을 아이와 함께 다녀본 것
3. 아이에게 무작정 화내지 않고 먼저 들어준 것
...
...
```

## 3. 나만의 원칙

### 구조
```
내가 [어떤 상황/경우에서] 지키는 원칙

첫번째 원칙: [기본 규칙]
두번째 원칙: [핵심 원칙]
세번째 원칙: [실천 원칙]
마지막 원칙: [결론적 원칙]

[짧은 마무리/통찰]
```

### 핵심 규칙
- 원칙은 4개 (첫번째~마지막)
- 각 원칙은 짧고 명확하게
- 마지막에 한 줄 마무리

### 예시
```
내가 악플러를 만났을 때 대응하는 법

첫번째 원칙: 악플은 무시하기
두번째 원칙: 차단 버튼 활용하기
세번째 원칙: 감정소모 최소화하기
마지막 원칙: 나의 시간 지키기

결국 나를 지키는 건 나 자신이에요.
```

## 4. YES/NO 템플릿

### 구조
```
[특정한 상황에 마주했을 때] 저는 이렇게 해요.

하지 말아야 할 것:
1. [금지사항 1]
2. [금지사항 2]

해야 할 것:
1. [필수사항 1]
2. [필수사항 2]

[결론적 조언]
```

### 핵심 규칙
- "하지 말아야 할 것" 2~3개
- "해야 할 것" 2~3개
- 마지막에 공감되는 조언 한 줄

### 예시
```
번아웃을 마주했을 때, 저는 이렇게 해요.

하지 말아야 할 것:
1. 번아웃을 무시하면서 계속 일하는 것
2. 나 자신을 소중히 하지 않는 것

해야 할 것:
1. 10일간 '돈을 안 벌어도 된다'고 각오할 것
2. 충분히 쉬지 않으면 길게 가지 못함을 다짐할 것

번아웃은 중요한 신호에요. 나 자신을 잘 챙겨야 오래갈 수 있어요.
```

## 5. 일상 인사이트

### 구조
```
오늘 [장소/상황]에서
문득 이런 생각이 들었어요.

[평범한 관찰 내용]

근데 이걸 가만히 보니까
[의외의 인사이트]가 보이더라고요.

그래서 [실천한 작은 변화]를 했어요.
그랬더니 [긍정적인 결과]가 생겼네요.

작은 관찰 하나가
이렇게 변화를 만드는 것 같아요.
```

### 핵심 규칙
- 관찰 → 인사이트 → 행동 → 결과 흐름
- 평범한 일상에서 출발
- 마지막은 "작은 관찰 하나가..." 패턴으로 마무리

### 예시
```
오늘 책상에 앉아서
문득 이런 생각이 들었어요.

제가 매일 똑같은 자리에서
컴퓨터를 켜놓고 휴대폰을 같이 바라보고 있더라고요.

근데 이걸 가만히 보니까
내가 '진짜 미디어 중독'이구나 싶더라고요.

그래서 둘 다 있을 때는
딱 하나만 하기로 했어요.

미디어를 이중으로 노출해서
나를 혹사시키지 않으려고 해요.

작은 습관 하나가
이렇게 변화를 만드는 것 같아요.
```

## 6. 실패 극복기

### 구조
```
[활동/시도] 실패 후
이렇게 극복했어요.

당시 상황은 이랬죠:
[구체적 상황]

이런 시도를 해봤어요:
1. [시도1]
2. [시도2]
3. [시도3]

전환점은 이거였어요:
[핵심 변화]

지금은 이렇게 됐네요:
[현재 상태]
```

### 핵심 규칙
- 실패 상황을 구체적 숫자로 (매출 0원, 구독자 3명 등)
- 시도한 것 3개 나열
- 전환점은 하나로 명확하게
- 현재 상태도 숫자로

### 예시
```
첫 온라인 강의는 대실패였답니다.

당시 상황은 이랬죠:
사전신청자 5명, 수강생 0명, 매출 0원

이런 시도를 해봤어요:
1. 무료 강의 등록 신청자 받기
2. 광고/SNS/까페 등 뿌릴 곳 다 뿌려보기
3. 커리큘럼을 알차게 개편하고 후기 전시

전환점은 이거였어요:
무료 강의 50분, 이후 10분동안 내 상품 세일즈

지금은 이렇게 됐네요:
한 강의당 10명 수강, 100만원 매출
```

## 7. 나만의 전략

### 구조
```
[플랫폼/서비스]로 돈 버는
방법을 공유할게요.

[핵심 원리]

이렇게 시작했어요:
1. [단계1]
2. [단계2]
3. [단계3]

주의할 점이 있어요:
[주의사항]
[실용적 조언]
```

### 핵심 규칙
- 핵심 원리를 먼저 간결하게
- 단계는 3개로 압축
- 주의할 점으로 신뢰감 형성

### 예시
```
제가 뉴스레터로 돈 버는 방법을 공유할게요.

꾸준히 주 1회 '유료급 아티클'을 무료로 배포하고,
구독자를 300명 모은 후,
그 300명에게 '내 상품'을 홍보하는 것. 이것이 다였어요.

첫 300명을 모으기 전까지
1. 주 1회 양질의 정보 제공
2. 뉴스레터를 50개 이상 오픈톡방에 공유하여 구독자 모음
3. 그렇게 모은 구독자에게 내 상품을 콘텐츠와 함께 홍보 (글 하단에 자연스럽게 소개)

주의할 점이 있어요.
처음에는 너무 비싸게 받지 않는게 중요해요.
```

## 8. 효율화 이야기

### 구조
```
[업무/활동]을 자동화한
방법을 공유할게요.

전에는 이랬어요:
[수동 프로세스]

이렇게 바꿨죠:
1. [자동화1]
2. [자동화2]
3. [자동화3]

효과가 대단했어요:
[긍정적 결과]
```

### 핵심 규칙
- Before/After 대비 명확하게
- 자동화 단계 3개
- 결과는 숫자로 (3시간→30분 등)

### 예시
```
해외 콘텐츠 수집을
자동화한 방법을 공유할게요.

전에는 매일 3시간씩 수동으로 번역했는데요.

이렇게 바꿨죠:
1. RSS Feed로 해외 뉴스 자동 수집
2. GPT로 초벌 번역
3. Claude로 교정 후 노션에 배포

효과가 대단했어요:
작업시간 3시간 → 30분으로 감소

(더 구체적 방법은 댓글에 계속)
```

## 9. 일잘러의 특징

### 구조
```
[직장/분야]에서 만난
최고의 일잘러의 특징이에요.

이런 점이 달랐어요:
1. [특징1]
2. [특징2]
3. [특징3]

특히 인상적이었던 건,
[핵심 포인트]

이분에게서 배운 건,
[주요 교훈]
```

### 핵심 규칙
- 특징 3개 나열
- "특히 인상적이었던 건" 으로 하이라이트
- "이분에게서 배운 건" 으로 교훈 마무리

### 예시
```
이전 직장에서 근무하다가 만난
최고의 일잘러들의 특징이 있었어요.

이런 점이 달랐어요:
1. 철저한 문서화가 습관
2. 주니어 의견 존중
3. 야근 없이 성과 창출

특히 인상적이었던 건,
완벽보다 실행을 중시하는 태도

이분에게서 배운 건,
일에 순수하게 진심인 모습이에요.
```

## 10. 책 인사이트

### 구조
```
[책제목]에서 배운 최고의 인사이트.

핵심 내용은:
[주요 메시지]

더 자세히 말하자면,
1. [적용1]
2. [적용2]
3. [적용3]

[구체적 변화를 통해 알아낸 나의 결론]
```

### 핵심 규칙
- 핵심 메시지를 한 문장으로
- 적용 포인트 3개
- 마지막은 "나의 결론"으로 개인화

### 예시
```
일류의 조건에서 배운 최고의 인사이트.
"초일류들은 빠르게 훔친다."

1. 성공하고 있는 인물을 끝까지 파헤치기
2. 내가 알아낸 방법을 모두 다 따라해보기
3. 이후 나만의 방식으로 발전시키기

'모방'이 나를 가장 빠르게 성장시키더라고요.
```

## 11. 시간 관리

### 구조
```
요즘 [특정 시간]에 저는
[독특한 나만의 일하는 방식]으로 일하고 있어요.

이렇게 하였더니
1. [변화1]
2. [변화2]
3. [변화3]

결과적으로,
[긍정적 변화 제안, 혹은 소감]
```

### 핵심 규칙
- 구체적 시간대 명시
- 독특한 방식 소개
- 변화 3개 나열
- 마지막에 감성적 마무리

### 예시
```
출근 후 아침 10시에 저는
예약 문자로 할일을 기록하고, 6시에 도착하도록 설정해요.

이렇게 하였더니
1. 내가 그날 할일을 다 했는지 알게 되고
2. 못한 일은 무엇이었는지 다시 파악하고
3. 쌓이다보니 점점 내 개선점을 알게 되더라고요

일종의 자동 회고 시스템이라고 할까요. :)
```
