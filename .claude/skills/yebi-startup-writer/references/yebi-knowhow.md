## 1. 정부지원사업의 본질 이해

### 1.1 정부지원사업이란?

**핵심 개념:**
- **대출(LOAN)이 아닌 지원금(GRANT)**: 갚지 않아도 되는 돈
- **단순 자금 지원이 아닌 성장 프로그램**: 사업의 방향 점검 + 실행력 향상
- **3가지 핵심 혜택**: 자금지원 + 교육·멘토링 + 신뢰도 확보

**정부가 지원하는 이유:**
```
고용 창출 → 일자리 증가
매출 증대 → 세금 증가 → 경제 활성화
산업 발전 → 국가 경쟁력 강화
```

**정부가 원하는 사업의 본질:**
> "이 사업이 왜, 누구의 어떤 문제를, 무엇으로, 어떻게 해결해 돈을 벌 것인가?"

### 1.2 정부지원사업을 해야하는 이유

**💰 1. 정부지원사업은 '순수익'과 같다**
- 융자가 아닌 100% 무상 지원금
- 제품 개발비, 마케팅비, 인건비, 디자인·특허비까지 지원
- 순수익 1,000만원을 벌려면 매출 3,000만원~1억원 필요
- 정부가 그만큼의 리스크와 비용을 대신 부담

**🏛 2. 단순한 '돈'이 아니라 '공신력'**
- 국가가 공식 인정한 기업이라는 신뢰
- 투자유치, 크라우드펀딩, 입점 심사, 브랜드 제휴에서 유리
- "정부지원 선정기업" 타이틀은 강력한 신용 신호

**🏢 3. 돈보다 큰 혜택: 인프라와 기회**
- 무상 사무실, 장비, 교육, 멘토링, 네트워킹
- 보육공간·마케팅 교육·전시 기회 제공
- 브랜드를 '진짜 사업'으로 성장시키는 발판

### 1.3 예비창업패키지 특징

**대상:**
- 사업자등록을 하지 않은 예비창업자
- 최대 7천만원 지원
- 아이디어 검증 및 시제품 제작 중심

**⚠️ 중요 포인트:**
- 공고 기준 **한 달 전까지 미창업 상태** 유지 필수
- 사업자 등록일 하나로 합격/탈락이 결정됨

### 1.4 정부가 보는 3가지 핵심 기준

```
1. 수익성: 제품/서비스의 실현 가능성 및 시장성
2. 실행력: 대표자 및 팀의 추진력, 전문성, 구체적 전략
3. 사회적 가치: 사회·환경·산업적 기여 효과
```

**균형이 중요:**
- 공공성만 강조 ❌
- 매출만 강조 ❌
- 세 가지의 조화 ✅

**정부가 기업을 지원하는 5가지 이유:**
1. 고용: 일자리 창출
2. 매출: 경제 활성화
3. ESG: 사회 문제 해결
4. 기술 혁신: 산업 경쟁력 강화
5. 투자 유치: 중소기업 육성

### 1.5 정부가 원하는 '솔루션'의 기준

**정부는 '완전히 새로운 기술'보다 시장 안에서 실제로 작동 가능한 해결책을 선호**

**지원 가능 vs 불가능 예시:**

| 불가능 | 가능 |
|--------|------|
| 부동산 투자 서비스 | 부동산 정보를 쉽게 비교하고 찾을 수 있는 서비스 (직방, 다방) |
| 1인 도매 쇼핑몰 | 사용자 취향에 따라 제품을 큐레이션하는 플랫폼 (지그재그) |
| 코인 리딩방 | 투자 교육이나 안전한 거래 습관을 돕는 서비스 |

**핵심 차이:**
> "대표 혼자 돈을 버는 게 아닌, 문제를 해결하는 구조여야 한다"

## 2. PSST 프레임워크 완벽 가이드

### 2.1 PSST 구조의 이해

```
P - PROBLEM (문제 정의)
  ↓ 무엇이 문제인지 명확히 알고 있는가?

S - SOLUTION (해결방안)
  ↓ 그것을 어떤 방식으로 해결할 것인가?

S - SCALE-UP (성장전략)
  ↓ 해결 이후 어떻게 성장할 계획인가?

T - TEAM (조직구성)
  ↓ 그걸 실행할 사람이 누구인가?
```

### 2.2 PSST가 중요한 이유

**심사위원의 평가 기준:**
1. 논리의 순서가 깔끔한가?
2. "아, 이 사업은 이해된다"라고 느껴지는가?
3. 실행 가능성이 보이는가?

**PSST = 정부가 사업을 평가하는 언어**

정부지원사업 심사위원은 하루에 수백~수천 개의 사업계획서를 검토합니다. 그 안에서 당신의 사업이 눈에 들어오려면:
- 논리의 순서가 깔끔해야 하고
- 읽는 사람이 "아, 이 사업은 이해된다"라고 느껴야 함

### 2.3 표준 사업계획서 구조

```
1. 창업아이템 개요 (요약) - 1~2페이지

2. 본문 (PSST) - 15~20페이지
   ├─ PROBLEM (문제 정의) - 3~4페이지
   ├─ SOLUTION (해결방안) - 5~7페이지
   ├─ SCALE-UP (성장전략) - 4~5페이지
   └─ TEAM (조직구성) - 2~3페이지
```

### 2.4 PSST 한눈에 보기

| 요소 | 핵심 질문 | 평가 기준 |
|------|-----------|----------|
| **PROBLEM** | 이 사업이 왜 필요한가? | 문제의 실재성 |
| **SOLUTION** | 무엇으로 해결할 것인가? | 실현 가능성 |
| **SCALE-UP** | 어떻게 성장시킬 것인가? | 확장성 |
| **TEAM** | 누가 실행할 것인가? | 전문성, 운영 가능성 |

## 3. PROBLEM (문제 정의) 작성법

### 3.1 핵심 원칙

**정부가 원하는 문제 정의:**
- ❌ "OO 산업은 성장 가능성이 높습니다"
- ❌ "OO 시장은 아직 경쟁이 적습니다"
- ✅ "고객이 실제로 겪는 현실적인 불편함"

**심사위원이 듣고 싶은 것:**
> "지금 사회에서 어떤 불편이 있고, 그걸 당신의 아이템이 어떻게 해결하느냐"

**심사위원은 감정이 아니라 데이터로 설득됩니다:**
- "이게 문제다" (X)
- "이 문제로 인해 얼마나 많은 불편이 발생하는가" (O)

### 3.2 PAIN POINT 정의 프레임워크

**6단계 구조:**

| 단계 | 핵심 질문 | 예시 |
|------|-----------|------|
| 1. 고객 세그먼트 | 누가? | 25~40세 초보 부모 |
| 2. 현재 사용 솔루션 | 무엇을 쓰고 있나? | 일반 포대기, 해외 수입 아기띠 |
| 3. 겪는 불편/불만 | 어떤 불만이 있나? | 장시간 사용 시 어깨·허리 통증, 아기 발달 불안 |
| 4. 불편 지속 시 문제 | 불편이 지속되면? | 양육 스트레스, 아기 건강 문제 |
| 5. 고객이 바라는 변화 | 진짜 니즈는? | 체중 분산 설계, 인체공학적 디자인 |
| 6. 해결 기회 | 우리가 해결할 부분? | 안전성과 편안함 동시 확보 |

### 3.3 합격하는 PAIN POINT의 4가지 특징

**1. 구체적이어야 한다**
```
❌ "많다, 불편하다"
✅ "국내 신생아 부모의 65%가 장시간 아기를 안을 때
    허리·어깨 통증을 호소합니다"
```

**2. 사회적 맥락이 있어야 한다**
```
❌ 개인의 불편
✅ "이 문제는 저출산 시대에 부모의 양육 부담을 키우는 원인 중 하나로,
    인체공학적 설계를 통한 포대기는 양육 스트레스를 줄이고
    출산율 정책에도 기여할 수 있습니다"
```

**3. 경제적 손실과 연결되어야 한다**
```
✅ "아토피 환아 가정은 연평균 100만 원 이상의 의료·관리비를 지출"
✅ "보호자 10명 중 7명이 기존 스킨케어 제품은 일시적 보습에만 그친다고 응답"
```

**4. 변화 방향이 명확해야 한다**
```
✅ "안전한 천연 성분 기반의 장기 사용 가능 제품"
✅ "체중 분산 설계, 아기 체형 고려한 인체공학적 디자인"
```

### 3.4 PAIN POINT 발굴 4단계 질문법

**Step 1: '누가' 가장 힘들어하고 있나?**
```
❌ "모든 소비자"
✅ "퇴직 후 1인 뷰티샵을 운영하는 40대 여성"
```

**Step 2: '언제' 불편을 느끼나?**
```
문제는 '순간'에 발생한다. 사용 과정, 거래 과정, 학습 과정 중 어느 타이밍?

예: "고객 관리할 때마다 고객별 시술기록이 뒤섞인다"
```

**Step 3: '왜' 이 문제가 반복되는가? (구조적 원인)**
```
제도, 비용, 기술, 인식, 접근성 — 어디서 막히는가?

예: "미용 CRM은 프랜차이즈용으로만 설계되어
     1인샵은 데이터 관리 불가"
```

**Step 4: '이 문제가 해결되면' 어떤 변화가 생기는가?**
```
문제 해결의 효과가 곧 시장성

예: "관리 이력 기반으로 고객 재방문율을 30% 높일 수 있다"
```

**💬 TIP:**
> PAIN POINT는 고객의 '감정적 불편'이 아니라,
> 시간·비용·효율의 손실로 측정 가능한 문제여야 함

### 3.5 업종별 PAIN POINT 예시

#### 예시 1: 1인 뷰티샵 사장

**PAIN POINT:**
- 고객 시술 기록이 수기로 관리돼, 누락·혼동 발생
- 신규 고객 확보를 위한 마케팅이 어렵고 비용이 높음
- 예약·결제·관리 시스템이 통합되지 않아 업무 비효율
- 프리랜서 디자이너·시술사 간 정보 교류 단절
- 온라인 후기·입소문 외에는 홍보 수단 제한적
- 1인샵 대상 전용 보험·법적 지원 제도 부족

**요약:** "운영 효율화 + 고객 데이터화 + 지속가능한 수익모델 부재"

**솔루션 예시:**
```
"1인 뷰티샵 전용 AI 고객관리 CRM 서비스"

HOW:
- 시술 기록·사진·피부 상태를 자동 저장 및 분석
- 재방문 알림, 제품 추천, 예약·결제 통합 기능
- 클라우드 기반 앱 연동 시스템
- 프랜차이즈 뷰티샵 대비 70% 낮은 이용료

한 줄 요약:
"1인 뷰티샵의 고객관리 비효율 문제를 해결하는
 AI 기반 CRM 서비스"
```

#### 예시 2: 1인 크리에이터

**PAIN POINT:**
- 콘텐츠 촬영 장비와 편집툴 구입비가 너무 높음
- 영상 편집·썸네일·마케팅을 혼자 다 해서 비효율
- 안정적인 수익구조 없이 광고 수입에만 의존
- 협업 기회나 커뮤니티 부재로 정보 공유 어려움
- 다국어 콘텐츠 제작 시 번역·자막 비용 부담
- 장기적으로 브랜드화(굿즈·클래스·IP 확장) 어려움

**요약:** "초기비용 부담 + 시스템·인프라 부족 + 수익 안정화 미비"

**솔루션 예시:**
```
"1인 크리에이터 공동제작 스튜디오 플랫폼"

HOW:
- 장비·공간·편집툴을 시간 단위로 공유
- AI 자막·썸네일 자동생성 기능 포함
- 지역 단위 공유 오피스형 스튜디오 구축
- 멤버십 기반 월 구독제 운영

한 줄 요약:
"1인 크리에이터의 장비비 부담과 작업 비효율을
 해결하는 콘텐츠 제작 플랫폼"
```

#### 예시 3: 아기 포대기 브랜드 (합격 사례)

**고객 세그먼트:** 25~40세 초보 부모

**현재 사용 솔루션:** 일반 포대기, 해외 수입 아기띠

**겪는 불편/불만:**
- 장시간 사용 시 어깨·허리 통증
- 아기 다리·허리 발달에 불안

**불편 지속 시 문제:**
- 보호자의 육아 피로 증가
- 양육 스트레스, 아기 건강 문제

**고객이 바라는 변화:**
- 체중 분산 설계
- 아기 체형 고려한 인체공학적 디자인
- 안전성과 편안함 동시 확보

**✅ 합격 예시:**
```markdown
국내 신생아 부모의 65%가 '장시간 아기를 안고 있을 때
허리·어깨 통증'을 호소합니다.

기존 포대기는 체중 분산 설계가 부족해, 보호자의 건강 문제뿐
아니라 아기 발달에도 부정적 영향을 줄 수 있습니다.

이 문제는 저출산 시대에 부모의 양육 부담을 키우는 원인 중
하나로, 인체공학적 설계를 통한 포대기는 양육 스트레스를
줄이고 출산율 정책에도 기여할 수 있습니다.
```

**❌ 불합격 예시:**
```
"제가 포대기를 써보니 불편했어요. 무겁고 안 예쁘더라고요."
→ '불편'이라는 단어만 있고, 객관적 근거 없음

"예쁜 디자인 포대기를 만들고 싶습니다."
→ 단순 취향 문제처럼 보여서 정부가 지원해야 할 이유 부족
```

#### 예시 4: 아토피 화장품 브랜드 (합격 사례)

**고객 세그먼트:** 영유아 자녀를 둔 부모, 성인 아토피 환자

**현재 사용 솔루션:** 시중 보습제, 병원 처방 연고

**겪는 불편/불만:**
- 보습 효과는 짧고
- 장기 사용 시 부작용·고비용 부담

**불편 지속 시 문제:**
- 피부 증상 악화
- 생활의 질 저하
- 의료비 증가

**고객이 바라는 변화:**
- 안전한 천연 성분 기반의 장기 사용 가능 제품
- 맞춤형 관리 솔루션

**✅ 합격 예시:**
```markdown
국내 소아의 약 20%가 아토피 피부염을 겪고 있으며,
보호자 10명 중 7명이 '기존 스킨케어 제품은 일시적
보습에만 그친다'고 응답했습니다.

이로 인해 아토피 환아 가정은 연평균 100만 원 이상의
의료·관리비를 지출하고 있습니다.

아토피는 만성 질환으로 사회·경제적 부담이 큰 만큼,
천연 성분 기반의 맞춤형 스킨케어는 건강보험 지출 절감과
K-바이오 헬스케어 산업 발전에 기여할 수 있습니다.
```

**❌ 불합격 예시:**
```
"저도 아토피가 있어서 피부가 가렵고 힘들었어요.
 그래서 브랜드를 만들고 싶었어요."
→ 개인적인 경험에만 국한

"요즘 아토피 환자가 많은데, 특별한 제품이 없는 것 같아요."
→ 수치·시장 데이터 없이 막연한 진술
```

#### 예시 5: 시니어 코딩 교육 플랫폼 (합격 사례)

**고객 세그먼트:** 55~65세 퇴직자·재취업 희망자

**현재 사용 솔루션:** 유튜브 무료 강의, 대학 평생교육원 강좌

**겪는 불편/불만:**
- 수업 속도가 너무 빠름
- 질문하기 어려움

**불편 지속 시 문제:**
- 학습 중도 포기
- 디지털 격차 확대
- 재취업 실패

**고객이 바라는 변화:**
- 눈높이에 맞춘 맞춤형 교육
- 실습·피드백 중심 수업

**✅ 합격 예시:**
```markdown
국내 50~60대 인구의 디지털 활용 역량은 2030세대 대비
40% 이상 낮으며, 고용노동부 조사에 따르면 시니어 구직자의
68%가 '디지털 역량 부족'을 가장 큰 취업 장애 요인으로 꼽았습니다.

하지만 기존 교육 프로그램은 청년층 위주라 시니어 맞춤형
커리큘럼은 매우 부족합니다.

시니어 대상 코딩 교육 플랫폼은 재취업, 평생교육,
디지털 격차 해소라는 세 가지 정책 목표에 기여할 수 있으며,
초고령 사회 진입에 따른 사회적 비용 절감에도 효과적입니다.
```

### 3.6 PAIN POINT 작성 팁

**BEFORE-DURING-AFTER로 나누기:**

```
구매 전: 정보가 너무 많아 선택이 어렵다
구매 중: 결제 과정이 복잡하거나 불안하다
구매 후: 기대한 만큼 효과를 느끼지 못한다
```

**⚠️ 피해야 할 문제 정의:**
```
❌ 감정적 진술만: "무겁고 안 예뻐요"
❌ 개인 경험만: "제가 써보니 불편했어요"
❌ 정부 지원 이유 불명확: "예쁜 걸 만들고 싶어요"
```

## 4. SOLUTION (해결방안) 작성법

### 4.1 SOLUTION 파트의 3요소

```
WHAT (무엇을 해결하는가)
  → 고객의 PAIN POINT 재강조

HOW (어떻게 해결하는가)
  → 기술적/운영적 방식, 서비스 구조

WHY BETTER (왜 더 나은가)
  → 기존 제품과의 차별성·효과성
```

**심사위원이 보는 것:**
> "이게 진짜 실행 가능한가?"

따라서 기술, 구조, 비즈니스 모델을 실현 가능성이 높게 보이도록 구체화해야 함

### 4.2 제품/서비스 한 줄 정의

**💡 왜 한 줄 정의가 필요한가?**
- 심사위원은 수백~수천 개의 사업계획서를 단시간에 검토
- "이 팀이 무엇을 하는지" 단 한 문장으로 즉시 이해시켜야 기억에 남음
- 사업의 본질을 짧고 명확하게 보여주는 브랜드 슬로건 같은 역할

**템플릿:**
```
제 아이템은 [타깃 고객]의 [문제]를 해결하는 [제품/서비스]입니다.
```

**✅ 한 줄 정의 예시:**
```
제 아이템은 맞벌이 부부의 식사 준비 시간 부족 문제를 해결하는
5분 완식 밀키트 브랜드입니다.

제 아이템은 소비자의 사이즈 교환·반품 불편을 해결하는
AI 가상피팅 서비스입니다.

제 아이템은 반려동물 보호자의 이동 스트레스 문제를 해결하는
체중별 맞춤 이동가방입니다.

제 아이템은 소상공인의 홍보 인프라 부족 문제를 해결하는
지역 기반 영상 콘텐츠 제작소입니다.
```

### 4.3 아이템 설명 4요소

심사위원이 '제품이 실제로 어떤 모습인지 머릿속에 그릴 수 있도록' 구체적으로 설명

**📁 아이템 설명 구조:**

**1) 형태 (FORM)**
```
크기, 재질, 구성요소, 외관 등 구체적 물리 정보

예: "책상 위에 올려두는 미니 디바이스 형태,
     15CM 높이, 알루미늄 재질"
```

**2) 기능 (FUNCTION)**
```
사용자가 실제로 할 수 있는 일, 사용자 경험 중심

예: "폐섬유 재질을 자동 분류하고 훼손율을 분석해
     등급별로 분류"
```

**3) 기술 (TECHNOLOGY)**
```
어떤 기술이 기반이 되는지, 기술적 차별성

예: "하이퍼스펙트럼 분석 기술을 활용해
     손상률을 정밀 측정(정확도 95%)"
```

**4) 디자인 (DESIGN)**
```
시각적 콘셉트, 색상, UI, 디자인 철학

예: "미니멀 디자인으로 일상 공간에 자연스럽게 배치"
```

**💬 TIP:**
> 심사위원은 "이 아이템이 실제로 생산/개발 가능한가?"를 중심으로 판단

### 4.4 핵심 기술 설명 5단계

**구조:**
```
1. 한 줄 요약
   → 기술적 핵심을 한 문장으로

2. 핵심 기술
   → 작동 원리를 객관적으로

3. 성능 지표
   → 숫자로 보여줄 수 있는 객관적 근거

4. 보호 장치
   → 특허, 인증, 보안 등

5. 확장 가능성
   → 데이터, API, 모듈 등으로 확장 방향
```

**실전 예시: 음수량 정밀 트래킹 스마트 보틀**

```markdown
### 한 줄 요약
다중센서 융합과 온디바이스 머신러닝으로 음료 종류·섭취량을
자동 인식하며, 방수·무선 연동까지 실사용 성능을 확보한
정밀 트래킹 보틀

### 핵심 기술

**다중센서 어레이**
- 전기전도도(EC), 온도(NTC), 이중파장 광학, 6축 IMU로
  액체 특성·기울기·음용 이벤트 동시 포착

**센서 융합 알고리즘**
- 노이즈 제거(모션·기포 보정) → 특징 추출 →
  경량화 분류기(온디바이스 ML)로 물/커피/탄산 등 자동 분류

**섭취량 측정 엔진**
- 정전용량 수위링 + IMU 기반 기울기 모델을 결합하여
  실시간 체적 변화 추정

**내구·방수 설계**
- 모듈 일체형 캡 구조, 이중 실링 가스켓, IPX7 등급 목표 설계

### 성능 지표
- 음료 분류 정확도: ≥93% (물/커피/탄산 3클래스, 상온 기준)
- 섭취량 측정 오차: ±3% (정지) / ±5% (일상 사용)
- 배터리 수명: 7일 이상 (일 100회 음용)
- 동기화 지연: <2초, 패킷 손실률 <0.5%
- 내환경성: IPX7 방수, 0~50°C 동작 범위

### 보호 장치

**지식재산:**
1. 음료 자동 분류 방법 (센서 융합·보정 로직)
2. 수위·기울기 결합 기반 체적 추정
3. 방수 일체형 센서 캡 구조
→ 특허 출원·등록 전략

**디자인:** 외관·UI 요소 디자인권 및 상표권 확보

**규제·인증:** 전자파·무선 적합(KC/FCC) 및 생활방수 시험

**보안:** 기기-앱 간 통신 AES 기반 암호화, 펌웨어 서명 검증

### 확장 가능성

**데이터 서비스**
- 영양·칼로리 추정, 주간/월간 리포트
- Apple Health 등 연동

**모듈러 에코시스템**
- UV 살균캡, 스포츠용 스트로우캡 등 액세서리 호환

**B2B 연계**
- 기업 웰니스 프로그램·보험사 인센티브 API 연동
- SDK 제공으로 서드파티 서비스 확장
```

**⚠️ 기술 설명 주의사항:**
```
❌ 'AI', '메타버스', '블록체인' 등 남발 금지
❌ 구체적 기술 설명은 최대 3페이지 넘지 말 것
✅ 기술은 "구현 가능성" 중심으로
✅ 수치는 '구체적'으로
✅ '단계별'로 나누면 이해가 쉬움
```

**💬 TIP:**
> 지금 쓰고 있는 것은 '사업'계획서이지, '연구'계획서가 아닙니다.
> 기술을 뽐내려다 본질을 잃지 마세요.
> 가장 중요한 것은 '어떻게' 사업화하여 돈을 벌 수 있는지 설득하는 것

### 4.5 창업 아이템 개발 진행 단계

**3단계 구조:**

```
🔍 아이디어 단계
- 문제를 구체적으로 정의하고, 고객의 실제 니즈 검증
- 고객 인터뷰 / 설문 / 커뮤니티 피드백
- 간단한 스케치, 콘셉트 모델 제작
- 시장 데이터 리서치로 실현 가능성 점검

🧩 시제품 개발 단계
- 아이디어를 실제 형태로 구현하고, 피드백 통해 개선
- 프로토타입 제작 (3D 프린팅, 간이 회로, 앱 등)
- 베타 테스트 (체험단, 소규모 고객) / 초기 KPI 달성도
- 알고리즘/디자인/기능 개선

💰 매출 단계
- 실제 고객이 돈을 지불하고 사용하는 단계
- 첫 판매, 크라우드 펀딩, 계약 체결
- 매출 지표 (재구매율, 전환율 등) 확보
- 투자/양산/해외 진출 준비
```

**💬 TIP:**
> 아이디어는 말로 끝나면 상상이고,
> 시제품과 매출이 검증되는 순간 비즈니스가 됩니다

**개발 진행 단계 예시: 음수량 트래킹 스마트 보틀**

```
아이디어 단계 (2023)
→ 아토피 환자, 다이어트 고객 100명을 인터뷰한 결과,
   70%가 실제 섭취 관리에 불편을 겪고 있음을 확인

시제품 개발 단계 (2024)
→ 현재 1차 프로토타입 완성, 체험단 30명 대상 테스트 중
→ 초기 수분 기록 정확도 70% → 알고리즘 개선 후 92% 향상

매출 발생 단계 (2025~)
→ 시제품 개선 후 크라우드 펀딩 목표 대비 200% 달성
→ 500명의 첫 유료 고객 확보
→ 현재 자사몰에서 정식 판매 중, 월 매출 1,000만원 달성
```

### 4.6 구체적인 실행 경험

**왜 중요한가?**
> "직접 해본 사람의 말"과 "머릿속에서만 한 말"은 다릅니다.

초기창업자에게 심사위원은 가능성보다 **실행력**을 봅니다.

**이렇게 써보세요:**

```
❌ "고객 조사를 진행했습니다"
✅ "20~40대 여성 100명을 인터뷰한 결과,
    78%가 사용 의향을 보였습니다"

❌ "시제품을 테스트 중입니다"
✅ "3D 프린터로 시제품을 제작해, 체험단에게
    2주 사용 피드백을 받았습니다"

❌ "펀딩을 계획하고 있습니다"
✅ "5월에 와디즈 프리오픈을 완료했고,
    사전 알림 신청자 700명을 확보했습니다"
```

**✅ 실행 경험 예시 (하나라도 있으면 유리):**
- 랜딩페이지 제작 / 사전 예약 진행
- 크라우드 펀딩 테스트
- 파일럿 프로젝트
- 체험단 / 무료 샘플링 운영
- B2B 제안서 / LOI 확보
- 초기 매출 / 첫 고객 확보
- SNS·커뮤니티 운영
- 공식 인증·심사 참여
- 협력사 MOU 체결

**💬 TIP:**
> 좋은 말보다 구체적인 숫자, 날짜, 행동이 훨씬 설득적
> 구체적인 시각 자료, 팀원이 나온 사진 등이 있으면 고득점

### 4.7 차별성 확보 전략 4단계

#### I. 시장 규모 (TAM·SAM·SOM)

```
TAM (Total Addressable Market)
→ 전체 시장 규모
→ "이 분야의 글로벌 시장 규모"

SAM (Serviceable Available Market)
→ 실제 진입 가능한 시장
→ 지역, 연령, 사용 환경 등 현실적 범위

SOM (Serviceable Obtainable Market)
→ 당장 점유할 수 있는 시장
→ 보통 SAM의 1~10% 정도로 잡으면 현실적
```

**예시:**
```
TAM: 전 세계 커피 시장 약 200조원
SAM: 국내 커피 시장 약 7조원
SOM: 디카페인 및 프리미엄 원두 소비 약 700억원
```

**요약:**
> TAM은 "비전", SAM은 "기회", SOM은 "현실"

#### II. 경쟁사 비교

**심사위원은 현존하는 솔루션과 어떻게 다른지 궁금해합니다.**

**비교표 예시: 아토피 비누**

| 구분 | 경쟁사 | 자사 |
|------|--------|------|
| **가격** | 개당 15,000원 내외 | 원료 직계약으로 12,000원대 |
| **품질** | 無향·無색소 마케팅, 평균적 보습 | 피부과 임상 테스트 완료, 보습 지속력 2배 |
| **유통/서비스** | 온라인몰, 로드샵 판매 | 피부과·약국 연계, 맞춤형 가이드 제공 |

**💬 TIP:**
> 임상, 제휴, 원가 절감 같은 객관적 근거로 비교하면 합격률 급상승

#### III. 자사 강점 찾아내기

**업종별 강점 템플릿:**

**🍔 식품 / F&B 브랜드**
```
기술: 저온 착즙·무첨가 공법으로 영양 손실 30%↓
원가: 지역 농가 직거래로 원가 15% 절감
네트워크: 로컬푸드 협동조합·백화점 PB 제휴
정책 연계: 농식품부 로컬푸드 사업, 친환경 인증
```

**👩🏻‍💻 IT 서비스 / 플랫폼**
```
기술: AI 맞춤 추천 알고리즘, 만족도 92%
원가: 자체 서버 구축으로 외주비용 40% 절감
네트워크: 대학 연구소·스타트업 협력
정책 연계: 중기부 비대면 바우처·스마트서비스 보급
```

**💄 뷰티 / 생활제품**
```
기술: 임상테스트 완료, 유해성분 無첨가
원가: 원료 직계약으로 생산비 20% 절감
네트워크: 피부과·약국·협회 연계
정책 연계: ESG·친환경 인증, 공공기관 납품
```

**📖 교육 / 콘텐츠**
```
기술: AI 분석 기반 개인 맞춤형 학습 시스템
원가: 자체 영상 제작으로 외주비용 절감
네트워크: 교사 커뮤니티, 대학생 멘토단
정책 연계: 교육부 디지털 학습지원 정책
```

**💬 TIP:**
> 자사 강점은 "기술 1줄, 수치 1개, 협력 1곳"만 있어도 신뢰도 급상승
> "우리가 잘한다"보다 "우리가 증명했다"는 톤으로 표현

#### IV. 초기 시장 진입 전략

**4단계 구조:**

**① 핵심 고객군 정의**
```
문제의 심각성이 높고,
해결책에 비용 지불 의사가 있으며,
주변에 입소문 전파력이 높은 집단을 초기 수용자로 설정
```

**핵심 고객군 3요소:**

| 타입 | 설명 | 예시 |
|------|------|------|
| 👨🏻 직접 사용자 | 제품/서비스를 직접 사용하는 사람 | 아토피 환아를 둔 부모 |
| 💰 구매 결정자 | 구매 결정에 영향을 주는 사람 | 장시간 근무로 수분 섭취 부족한 직장인 |
| 🏢 영향력 있는 전문가/기관 | 전문성으로 신뢰를 주는 집단 | 피부과 전문의, 기업 복지 담당자 |

**② 유통 채널 선정**

```
🛒 직접 판매 (D2C)
- 자사몰, 오프라인 매장, 모바일 앱 중심
- 고객 경험과 브랜드 통제력 강화
- 초기 고객 피드백 직접 수집, 즉시 개선 반영
→ 빠른 검증과 브랜드 신뢰도 형성 단계

🌐 온라인 플랫폼
- 쿠팡, 네이버, 오늘의집, 폐쇄몰 등 다채널 입점
- 플랫폼 내 광고·리뷰 시스템 활용
- SNS 콘텐츠 연계한 판매 구조
→ 대중 인지도 및 매출 확대 단계

🏛 B2B / B2G 채널
- 기업 복지몰, 학교, 병원, 정부 조달
- 공공기관·의료기관 협력을 통한 신뢰 기반 유통
- ESG, 사회적 가치, 인증 기반 제품일 경우 유리
→ 안정적 매출 및 레퍼런스 확보 단계
```

**채널 선정 체크리스트:**
```
□ 고객 접근성 – 실제 구매가 일어나는 채널인가?
□ 비용 효율성 – CAC 대비 LTV가 적절한가?
□ 신뢰도 – 브랜드 인지도 보완이 가능한가?
□ 속도 – 검증까지 걸리는 시간이 짧은가?
□ 확장성 – 향후 시장 확장 시 발판이 될 수 있는가?
□ 실현 가능성 – 실제 계약/입점이 가능한가?
```

**💬 도비의 팁:**
> 유통 채널은 '많이'보다 '효율적'이 중요
> "가장 빨리 반응이 오는 채널부터 → 신뢰도 높은 채널로 확장"

**③ 마케팅 전략**

**신뢰 기반 마케팅 4요소:**

```
1. 전문가·기관 검증
   - 인증서, 임상 시험, 특허 등 객관적 신뢰 확보
   - 피부과·병원·공공기관 등과 협력
   - "좋다고 말하는 브랜드"가 아닌 "검증된 브랜드"

2. 파일럿 및 체험단 운영
   - 소규모 실증 테스트로 제품 신뢰도·효과성 검증
   - 초기 소비자 후기와 피드백 수집
   - 체험단 리뷰는 광고보다 전환율 높은 자발적 마케팅

3. 객관적 증거 확보
   - 사용 전후 비교 사진, 설문 통계, 재구매율
   - 리뷰·임상 리포트·데이터로 설득력 강화

4. 파트너십 및 제휴
   - 병원·공공기관·전문 협회 협업
   - 브랜드 신뢰도 및 사회적 공신력 확보
   - 공공 캠페인, ESG 프로젝트 연계
```

**채널별 시장 진입 전략:**

| 채널 | 목적 | 전략 |
|------|------|------|
| **디지털 마케팅** | 유입 확보, 빠른 데이터 수집 | 검색광고, SNS 광고, 이메일 캠페인 |
| **콘텐츠 마케팅** | 자연스러운 신뢰 유도 | 블로그·카페 후기, 숏폼 영상, 체험 콘텐츠 |
| **오프라인 마케팅** | 직접 체험 통한 신뢰 형성 | 팝업스토어, 체험 행사, 제휴 매장 입점 |
| **PR & 협업** | 사회적 신뢰와 노출 확대 | 언론보도, 기관 공동 캠페인, 브랜드 콜라보 |

**💬 TIP:**
> 심사위원이 주목하는 건 "광고 예산"이 아니라 "검증 과정"
> 실제 사용자 리뷰, 기관 협업, 임상 데이터 등 신뢰의 근거를 보여주는 사례가 핵심

**④ KPI 데이터 확보**

**📊 마케팅 및 고객 반응 지표:**
```
인지도/참여도: SNS 노출 수, 검색량, 콘텐츠 반응률
조회·노출수/클릭률: 광고·콘텐츠 효율성 측정
전환율/유지율: 방문자가 실제 구매로 이어지는 비율
구매율: 체험단 → 구매, 재구매 전환 등

핵심: "얼마나 많은 사람이 봤는가"보다
      "얼마나 많이 '행동'했는가"
```

**💰 매출 및 성과 지표:**
```
매출 / 장바구니 전환율: 유입 대비 구매 행동 비율
CAC (Customer Acquisition Cost): 고객 1명 유입당 마케팅 비용
LTV (Life Time Value): 고객 1명이 평생 남기는 순이익
NPS (Net Promoter Score): 추천 의향 기반 고객 충성도

핵심: ROI 기반으로 "마케팅 효율성을 수익화"
```

**🧠 기술 및 제품 성과 지표:**
```
특허 출원/기술 개발: 기술 경쟁력 및 진입장벽 확보
구독 유지율/페이지 체류시간: 제품 사용성·만족도 평가
데이터 기반 개선: 피드백 → 개선 → 재구매율 상승의 선순환

핵심: 기술적 신뢰성과 시장 적합성을 데이터로 증명
```

**💬 TIP:**
> 심사위원은 매출보다 데이터 기반 실행력과 학습 속도를 봄
> 초기에 모든 지표 다 잡기보다, 핵심 3개 KPI부터 명확히 측정

## 5. SCALE-UP (성장전략) 작성법

### 5.1 성장전략의 4가지 구성요소

```
① 비즈니스 모델
   → 제품 → 브랜드 → 플랫폼 구조로 확장

② 단계별 시장 진입 전략
   → 초기 타깃 → B2B → 글로벌 확장

③ 중장기 로드맵
   → 예비 → 초기 → 도약 단계별 목표

④ 사회적 가치 도입
   → ESG, 친환경, 지역상생
```

### 5.2 비즈니스 모델

**비즈니스 모델이란?**
> 저의 사업은 [어떤 고객]에게 [어떤 가치]를 [어떤 방식으로] 제공하고,
> 이를 통해 [어떻게 돈을 버는지]를 구조화한 것

**비즈니스 모델 캔버스 9요소:**

```
┌─────────────┬─────────────┬─────────────┬─────────────┬─────────────┐
│핵심 파트너  │  핵심 활동  │             │  고객 관계  │  목표 고객  │
├─────────────┤             │  가치 제안  ├─────────────┤             │
│  핵심 자원  │             │             │  유통 채널  │             │
├─────────────┴─────────────┴─────────────┴─────────────┴─────────────┤
│                비용 구조                  │         수익 구조          │
└───────────────────────────────────────────┴────────────────────────────┘
```

**각 요소별 설명:**

**1) 핵심 파트너 (KEY PARTNERS)**
```
제품/서비스 개발, 생산, 유통, 기술, 마케팅에서 협력하는
주요 파트너 또는 기관

예: 원료 공급사, 제조 협력사, 유통 파트너, 연구소,
    협회, 자문단
```

**2) 핵심 활동 (KEY ACTIVITIES)**
```
비즈니스 운영을 위해 가장 중요한 활동

예: 연구개발, 품질 관리, 마케팅, 고객 관리, 제조,
    플랫폼 운영
```

**3) 핵심 자원 (KEY RESOURCES)**
```
사업의 기반이 되는 자원

예: 인력, 기술, 브랜드, 지식재산, 설비, 데이터,
    네트워크
```

**4) 가치 제안 (VALUE PROPOSITION)**
```
고객이 이 제품/서비스를 선택할 이유

예: 문제 해결, 효율성, 편의성, 감성적 가치,
    가격 경쟁력
```

**5) 고객 관계 (CUSTOMER RELATIONSHIP)**
```
고객과 관계를 유지하는 방식

예: 구독 서비스, 커뮤니티, 개인화 서비스,
    자동화 서비스, 전담 지원
```

**6) 유통 채널 (CHANNELS)**
```
고객에게 제품/서비스를 전달하는 경로

예: 온라인몰, 앱스토어, 오프라인 매장, 납품,
    파트너 채널
```

**7) 목표 고객 (CUSTOMER SEGMENTS)**
```
가장 중요한 고객군의 특징과 니즈

예: 2030 여성, 1인 가구, 중소기업, 시니어 세대
```

**8) 비용 구조 (COST STRUCTURE)**
```
고정비·변동비를 포함한 주요 비용 항목

예: 인건비, 마케팅비, 서버비, 제조비, 임대료,
    물류비
```

**9) 수익 구조 (REVENUE STREAMS)**
```
수익을 창출하는 방식

예: 판매, 구독, 광고, 수수료, 라이선스, 데이터 판매
```

**비즈니스 모델 작성 체크:**
```
✅ 고객 정의: 누가 돈을 지불하는가?
✅ 수익 구조: 구독·수수료·판매 등 핵심 모델 1~2개만 명확히
✅ 차별 포인트: 기술이 아닌 돈의 흐름으로 설명
✅ 숫자 제시: 시장규모·예상매출 등 근거 있는 수치
```

### 5.3 단계별 시장 진입 전략

**3단계 진입 전략:**

```
1단계: 시장 검증 (초기 1~3개월)
├─ 목적: 초기 고객 확보 및 테스트
├─ 전략: 베타테스트, 체험단, 파일럿 프로젝트
└─ 핵심: 시장 반응 확인 중심

2단계: 성장 (3~6개월)
├─ 목적: 본격적인 판매 개시 및 인지도 확보
├─ 전략: 온라인·오프라인 병행, 협업/파트너십 확장
└─ 핵심: 재구매 구조 강화

3단계: 확장 (~1년차)
├─ 목적: 브랜드 확장, 신규 타깃 진입
├─ 전략: 해외 진출, B2B 납품, 제품군 확대
└─ 핵심: 수익 모델 다변화
```

### 5.4 중장기 로드맵

**3단계 시간축 구조:**

```
📍 단기 (1년 이내)
⚙ 제품·기술 목표: 제품 완성도 향상, 기술 고도화 착수
📊 시장·매출 목표: 초기 매출 발생, 고객 피드백 확보
👥 조직·운영 목표: 핵심 인력 확보, 파트너 네트워크 구축

📍 중기 (2~3년)
⚙ 제품·기술 목표: 라인업 확장, 품질 안정화 및 인증 확보
📊 시장·매출 목표: 시장 점유율 확대, 재구매율 30% 이상 달성
👥 조직·운영 목표: 전담 부서 신설, 내부 R&D 역량 강화

📍 장기 (4~5년)
⚙ 제품·기술 목표: 기술 특허 및 독자 플랫폼 구축
📊 시장·매출 목표: 해외 진출 및 대형 유통망 확보
👥 조직·운영 목표: 생산·유통 자립화, ESG 경영 기반 확립
```

**작성 원칙:**

```
✅ "성과 중심" 객관적 지표
   → '준비', '시작'이 아닌 '○○ 달성', '○○ 구축 완료'
   → 매출, 점유율, 고객 수 등 수치 중심 목표

✅ "시각화"
   → 연도별 표, 타임라인 그래프 형태로 한눈에 이해 가능하게

✅ "장기적 비전 한 줄"
   → "5년 내 업계 표준을 선도하는 기술 기반 기업으로 성장"
```

**⚠️ 로드맵 작성 시 유의점:**

```
❌ 너무 소극적인 목표
   → 리스크를 두려워하면 도전정신 부족해 보임

❌ 허황된 숫자
   → 근거 없는 '3년 내 매출 500억'은 오히려 신뢰도↓

✅ '살짝 높은 목표'
   → 도전적이지만 구체적, 실행력 있는 대표자로 보임
```

**💬 TIP:**
> 중장기 로드맵은 사업계획서의 마지막이지만,
> 심사위원에게 "이 사업을 믿고 지원해도 되겠다"는 확신을 주는 핵심 파트
> 당장의 수익보다 방향성과 실행력을 명확히 보여주는 것이 중요

### 5.5 사회적 가치 도입

**🌎 사회적 가치란?**
> 기업이 단순히 이익을 창출하는 것을 넘어,
> 환경(E), 사회(S), 지배구조(G) 측면에서
> 사회 전체에 긍정적 영향을 미치는 활동

**ESG 3가지 핵심 영역:**

```
🌱 환경 (E)
- 친환경 생산, 자원 절감, 탄소 저감
- 예: 재활용 소재 활용, 리필 시스템, 탄소중립 인증

👥 사회 (S)
- 고용 창출, 지역사회 환원, 소비자 안전
- 예: 취약계층 고용, 청년 창업 지원, 안전한 제품 개발

🏢 지배구조 (G)
- 투명한 경영, 윤리경영, 내부 관리체계
- 예: 경영 정보 공개, 윤리 헌장 수립, 반부패 정책
```

**3가지 도입 방향:**

```
🔹 내재형 (사업 내부에서 실현되는 가치)
→ 제품/서비스 자체가 사회문제 해결
→ 예: "폐플라스틱을 재활용해 환경오염을 줄이는
       업사이클링 화분"

🔹 운영형 (운영 과정에서의 사회 환원)
→ 사업 수익 일부를 사회에 재투자
→ 예: "청년 창작자와의 협업을 통해 지역 일자리 창출"

🔹 확산형 (기업 영향력을 통한 사회 캠페인)
→ 브랜드 메시지를 활용해 사회적 인식 확산
→ 예: "환경보호 메시지를 담은 캠페인 제품 판매 및
       수익 환원"
```

**정부지원사업과의 연계 포인트:**

정부는 '잘 파는 기업'보다 '사회 문제를 해결하면서 수익을 창출하는 기업'을 선호

```
✅ 사업이 해결하는 사회·환경적 문제는 무엇인가?
✅ 그 효과가 수치나 구조로 증명 가능한가?
✅ 매출 성장과 사회적 가치가 동시에 확대되는 구조인가?
```

**💬 TIP:**
> "기업의 선행"이 아니라 "사업의 본질 속 사회적 가치"를 강조
> '지속가능한 사업 모델'로 평가받을 수 있도록 구조화

## 6. TEAM (조직구성) 작성법

### 6.1 조직 구성의 핵심

**💬 TIP:**
> "좋은 아이템보다 중요한 건 아이템에 맞는 경력의 팀과 보완할 네트워크"

**PSST의 마지막 'T'는:**
- 팀의 전문성
- 실행력
- 네트워크를 증명하는 구간

**✅ 조직 구성 핵심 4요소:**

```
🧑🏻‍💼 대표자 역량
→ 전공, 경력, 자격증, 수상, 창업 경험 등
→ "이 사업을 할 자격이 있는 사람인가?"

👬🏻 핵심 인력
→ 기술, 디자인, 마케팅 담당 등
→ "서로의 부족함을 채워줄 수 있는 조합인가?"

🎓 외부 네트워크
→ 자문단, 협력사, 멘토, 투자자, 파트너 기업
→ "혼자 하는 게 아니라, 이미 주변 자원이 있다"

📅 향후 채용 계획
→ 현재 인력 부족해도, "누구를 언제 보강할지" 명확하면 가점
```

**🚫 불합격 VS ✅ 합격 표현:**

```
❌ "사명감과 열정을 가지고 있습니다"
   → 감정적, 추상적 표현

✅ "삼성전자 5년 근무, 관련 특허 2건 보유"
   → 객관적, 구체적 근거
```

### 6.2 대표자 역량 4요소

```
1. 전공/경력
   → "이 사업과 나의 연결고리"
   → 사업 아이템과 관련성 높은 학력·경력부터 강조

   예: "산업디자인 전공, 가구 제조사 근무 3년"

2. 성과
   → "실행 결과가 있는 사람인가?"
   → 숫자·기간·성과를 구체적으로 (정량화 필수)

   예: "2023년 크라우드 펀딩 목표 250% 달성,
        디자인 어워드 수상"

3. 전문성
   → "기술, 인증, 자격으로 증명 가능한가?"
   → 특허, 논문, 자격증, 전문 프로젝트 참여 이력

   예: "3D 모델링 전문 자격, 센서 기술 특허 1건 보유"

4. 비전
   → "이 대표가 끝까지 갈 이유가 있는가?"
   → 진정성 있는 창업 동기와 장기적 목표를 간결히

   예: "어린 시절 피부 질환으로 고생하며,
        지속 가능한 헬스케어 솔루션 개발을 목표로 함"
```

### 6.3 대표자 전문성 보완 전략

**① 직접 경험으로 증명하기**

전문 전공이 아니더라도, '직접 부딪혀 배웠다'는 것은 강력한 신뢰 포인트

```
✅ 개발 전공은 아니지만 코딩 학원 6개월 수료
✅ 화학 전공은 아니지만 화장품 책임판매관리자 자격 취득
✅ 영상 전공은 아니지만 유튜브 실버버튼 보유
✅ 마케팅 전공은 아니지만 쇼핑몰 매출 10억 달성
✅ 요식업 전공은 아니지만 부모님께 30년 노하우 전수
```

**→ "이 사업을 위해 시간을 투자했다"는 증거**

**② 외부 전문가와 협업하기**

모든 걸 혼자 다 할 필요는 없습니다. 전문가를 관리하고 방향을 제시할 수 있다면 그것이 리더십

```
✅ 디자인 전공 대표
   → 공대 출신 CTO와 공동창업 (기술 부분 보완)

✅ 기술자 대표
   → 마케팅 CMO와 협업하여 브랜드 전략 수립

✅ 회계 지식 부족
   → 외부 세무법인 계약 및 보고 체계 구축
```

**핵심:**
> "부족한 부분을 인식하고, 누구와 함께 보완했는지를 명확히 보여주는 것"

**③ 분야별 보완 구조 제시**

| 분야 | 업무 | 보완 방법 | 예시 |
|------|------|-----------|------|
| **기술/전문성** | 제품/서비스 기획·개발, 품질 관리, 특허/인증 | CTO, 외부 전문가, 연구소 협력 | IT 대표(디자인 전공) + 공대 CTO → 기술 특허는 CTO 담당 |
| **영업/마케팅** | 고객 조사, 홍보, 브랜딩, 온오프라인 유통 | CMO, 외부 대행사, 외주 마케터 | 기술자 대표 + 브랜드팀 CMO → 타겟 세분화 및 채널 전략 |
| **재무/운영** | 자금 관리, 회계/세무, IR 피칭, 네트워킹 | CFO, 세무사, 행정사, 액셀러레이터 | 외부 세무법인 계약, 액셀러레이터 멘토단 IR 코칭 |

### 6.4 정부가 원하는 기업 조직 구성

**이상적인 조직 구조:**

```
           CEO (총괄 기획, 투자 유치)
            |
    ┌───────┼───────┬───────┐
    |       |       |       |
   CTO     CMO     CFO     COO
  (기술) (마케팅) (재무)  (운영)
```

**역할별 구성:**
```
CEO: 총괄 기획, 투자 유치, 대외 업무, 전략 수립
CTO: 기술 개발, 제품 설계, R&D, 품질 관리
CMO: 마케팅, 브랜딩, 고객 획득, 콘텐츠 전략
CFO: 재무 관리, 자금 조달, 회계, IR
COO: 운영, 생산, 품질 관리 (필요시)
```

**완성된 조직 구성 예시:**

```
CEO (대표자)
├─ 역할: 총괄 기획, 투자 유치
├─ 경력: OO대학교 경영학과 졸업
└─ 강점: 시장 분석, 자금 조달

핵심 팀원 1: CTO
├─ 이력: 화학공학 석사
└─ 역할: 임상/연구개발

핵심 팀원 2: CMO
├─ 이력: 대기업 마케터 5년
└─ 역할: 온라인 마케팅, 브랜딩

외부 네트워크
├─ 피부과 전문의 자문
├─ 제조 협력사
├─ 기술 개발사
├─ 액셀러레이터 멘토단
└─ 특허 자문단

채용 계획
├─ 상반기: 물류 관리자 1인
└─ 하반기: 제품 디자이너 1인
```

**💬 TIP:**
> 초기창업 단계에서는 모든 분야 인력을 완벽히 갖추기 어려움
> 하지만 중요한 건 부족함을 인지하고, 이를 보완할 계획을 갖고 있는가
> 기술·마케팅·운영 등 핵심 분야는 외부 네트워크 협업 또는 향후 채용 계획으로 보완

## 7. 사업계획서 완성도 높이기

### 7.1 가독성 향상 원칙

**왜 '가독성 있게' 다듬어야 하나?**
- 심사위원은 한 사업계획서를 3분도 채 읽지 않음
- 좋은 아이템이라도, 읽기 어렵다면 전달되지 않음
- 결국 '읽히는 사업계획서'가 선정됨

**읽히는 문서의 4대 원칙:**

```
① 시각 구조 (LAYOUT)
   → 문단 길이 줄이고, 표·도식 활용

② 논리 흐름 (FLOW)
   → 문제 → 해결 → 실행 → 성과의 자연스러운 흐름

③ 시각 요소 (VISUAL)
   → 불필요한 텍스트보다 도표·이미지 활용

④ 언어 톤 (TONE)
   → 명확하고 간결하게, 수치와 근거 위주
```

### 7.2 사업계획서 체크리스트

**📄 문서 구조·형식 점검**
```
□ 제목, 소제목의 계층 구조가 한눈에 들어오는가?
□ 본문 글자 크기 9~11PT, 줄 간격 140~180% 수준으로 읽기 편한가?
□ 정렬, 들여쓰기, 여백 등 전체 포맷 일관성이 유지되는가?
```

**📍 가독성·논리 흐름 점검**
```
□ "1분 안에 핵심이 보이는 문서"인가?
□ 각 항목이 자연스럽게 연결되고, 불필요한 반복이 없는가?
□ 구어체가 아닌 비즈니스 톤(보고서체)으로 작성되었는가?
```

**📍 데이터·레퍼런스 검증**
```
□ 모든 수치와 근거에 출처(기관명, 연도)가 명시되어 있는가?
□ 3년 이내의 최신 통계 데이터를 사용했는가?
□ 지나치게 과장된 예측이나 근거 없는 수치는 없는가?
□ 그래프, 표, 인포그래픽이 실제 데이터와 일치하는가?
```

**📍 시각적 구성·표현**
```
□ 핵심 데이터가 표, 그래프 등으로 시각화되어 있는가?
□ 색상, 폰트, 강조 표현(굵기·밑줄 등)이 일관적인가?
□ 해상도가 낮거나 깨진 이미지가 없는가?
□ 문서의 컬러톤이 브랜드 아이덴티티와 맞는가?
```

**📍 내용 완결성·일관성**
```
□ 문제 → 해결 → 성장 → 팀의 흐름이 자연스러운가?
□ 매출, 시장 규모 등 수치가 앞뒤 문맥과 일치하는가?
□ 중복되거나 모순된 내용이 없는가?
□ 마지막 결론이 '문제 정의'와 연결되어 마무리되는가?
```

### 7.3 요약문 작성법

**요약문 = 사업계획서의 얼굴**
> 심사위원이 가장 먼저 – 혹은 유일하게 끝까지 읽는 부분

**✍ 요약문 작성 원칙:**

```
🔹 이해도: 1분 안에 핵심 파악 가능
🔹 길이: 1~2페이지 이내 (A4 기준)
🔹 핵심 요소: 누가(팀), 왜(문제), 어떻게(해결), 어디로(성장)
🔹 시각화: 그래프, 표, 핵심 이미지 최소 1개 이상
🔹 언어: 짧고 단정한 문장 ("~입니다", "~을 통해")
```

**작성 TIP:**
```
✅ 첫 문장은 "무엇을, 왜, 누구를 위해" 명확히
✅ 각 항목별로 핵심 키워드(문제-해결-성과) 반복
✅ 시각자료는 "핵심 구조 요약" 중심으로
✅ 숫자는 3개 이하만 강조 (매출 목표, 고객 수, 효과 지표)
```

**요약문 구성:**
```
1. 아이템 개요 (이미지 포함)
2. 문제 인식 (PROBLEM) - 핵심 PAIN POINT
3. 실현 가능성 (SOLUTION) - 해결 방안과 차별성
4. 성장 전략 (SCALE-UP) - 비즈니스 모델과 로드맵
5. 팀 구성 (TEAM) - 전문성과 실행력
```

### 7.4 발표자료 구성

**발표는 보통 10분 이내, 슬라이드 15~20장 기준**

**핵심 구조:**
```
❓ 문제 정의
   → 고객의 PAIN POINT가 명확한 지표나 근거를 갖추고 있는가?

💡 해결 방안
   → 제품·서비스의 핵심 기술을 사용자 중심으로 이해하기 쉽게 설명했는가?

⚡ 차별성
   → 현재 시장 서비스를 제대로 알고, 이에 대한 강점을 제시했는가?

📈 성장 전략
   → BM, 시장 진입, 확장 계획이 단계적으로 구성되어 있는가?

👥 팀 소개
   → "이 팀이 이걸 꼭 해야 하는 이유"가 드러나는가? (전문성과 실행력)
```

**💬 도비의 팁:**
> 슬라이드보다 "논리 흐름"이 더 중요
> '왜 이 문제인가 → 그래서 어떤 해결책인가 → 그게 왜 우리인가'
> 이 3단 구조만 유지하면 발표는 흔들리지 않음

**발표 내용 점검:**
```
✍ 문장: "저희는 ~입니다" 단정하고 간결한 어조
📊 수치: 시장 규모, 매출 목표, 고객 수 등 명확한 수치
📋 사례: 사용자 후기, 실증 테스트 등 구체적 근거
🔗 논리: 각 문장 간의 흐름이 자연스럽고 연결성 있게
⏱ 시간: 한 슬라이드당 20~30초 내 전달 가능해야 함
```

**💬 도비의 팁:**
> "감성적 어필"보다 "이성적 설득"이 우선
> 수치 → 근거 → 결론 → 자신감, 이 순서로 말하면 전달력 배가

### 7.5 질의응답(Q&A) 대응

**발표 점수의 절반 이상은 Q&A에서 갈립니다**

**준비 사항:**

```
❓ 예상 질문 준비
   → "시장 검증?", "매출 근거?", "실행 가능성?" 등
      주요 질문 사전 대비

💬 답변 방식
   → "YES, BUT" 화법으로 대답
   → 예: "심사위원님께서 말씀하신 부분도 맞습니다. 하지만~"

📈 근거 제시
   → 수치, 테스트 결과, 고객 피드백 등 구체적 데이터로 대답

😎 태도
   → 방어적이지 않고 논리적으로, 끝까지 차분하게 설명

🙏 마무리
   → "좋은 의견 감사합니다. 반영해 개선하겠습니다"
      긍정적 피드백으로 마무리
```

**자주 나오는 질문과 답변 전략:**

```
Q1. "시장 검증은 어떻게 했나요?"
A: "고객 인터뷰 100명 진행, 체험단 30명 운영,
    사전 예약 700명 확보" (구체적 수치)

Q2. "경쟁사 대비 차별점은?"
A: [비교표 제시] "가격 20% 낮고, 성능 2배,
    피부과 연계 서비스 제공"

Q3. "매출 목표 근거는?"
A: "목표 고객 1,000명 × 객단가 50,000원 ×
    재구매율 30% = 연매출 1억 5천"

Q4. "팀 구성이 부족해 보이는데?"
A: "현재 CEO+CTO 구성, 외부 마케팅 대행사 협력,
    하반기 CMO 1인 채용 예정"

Q5. "기술 실현 가능성은?"
A: "프로토타입 완성, 30명 테스트 결과 정확도 92%,
    OO대학교 연구소와 기술 협력 중"
```

**💬 도비의 팁:**
> 심사위원이 기억하는 건 "답변 내용"이 아니라 "태도"
> 자신감 있게 답하는 태도가 실제 점수에 큰 영향

## 8. 실전 작성 체크리스트

### 8.1 단계별 작성 순서

**Step 1: 정보 수집 및 분석**
```
□ K-Startup에서 공고 확인
□ 지원 자격 요건 체크 (업력, 나이, 지역, 업종)
□ 제출 서류 양식 다운로드
□ 시장 데이터 수집 (3년 이내 통계)
□ 경쟁사 분석 (최소 3개 이상)
```

**Step 2: PAIN POINT 정의**
```
□ 고객 세그먼트 명확화
□ 현재 사용 솔루션 및 한계 파악
□ 구체적인 불편/불만 수치화
□ 경제적 손실 계산
□ 사회적 맥락으로 확장
```

**Step 3: 솔루션 구조화**
```
□ 한 줄 정의 작성
□ 형태·기능·기술·디자인 설명
□ TAM·SAM·SOM 시장 규모 산정
□ 경쟁사 비교표 작성 (3개 이상 항목)
□ 자사 강점 3가지 이상 도출
□ 개발 진행 단계 정리
□ 실행 경험 증명 자료 준비
```

**Step 4: 성장 전략 수립**
```
□ 비즈니스 모델 캔버스 9요소 작성
□ 3단계 시장 진입 전략 수립
□ 중장기 로드맵 작성 (1년/3년/5년)
□ 사회적 가치 (ESG) 도입 방안 포함
□ 자금 계획 (정부지원금 + 기타 조달)
```

**Step 5: 팀 구성 정리**
```
□ 대표자 역량 4요소 작성
□ 핵심 인력 역할 및 경력 정리
□ 외부 네트워크 리스트업 (3개 이상)
□ 향후 채용 계획 수립
□ 분야별 보완 구조 제시
```

**Step 6: 문서 완성**
```
□ 요약문 작성 (1~2페이지)
□ 본문 PSST 구조 완성 (15~20페이지)
□ 시각 자료 삽입 (표, 그래프, 이미지 10개 이상)
□ 전체 가독성 체크
□ 맞춤법·띄어쓰기 검사
```

**Step 7: 발표 준비**
```
□ 발표 자료 15~20장 구성
□ 10분 발표 리허설 (타이밍 체크)
□ 예상 질문 10개 이상 준비
□ Q&A 답변 시나리오 작성
□ 발표 태도 및 자신감 점검
```

□ 3년 이내 최신 통계 인용

```

□ 형태·기능·기술·디자인 설명
□ TAM·SAM·SOM 시장 규모
□ 경쟁사 비교표 (3개 이상 항목)
□ 자사 강점 3가지 이상
□ 초기 시장 진입 4단계 전략
□ 개발 진행 단계 명시

**TEAM 파트 필수 요소:**

□ 대표자 비전 (창업 동기)
□ 핵심 인력 역할 및 경력

### 8.3 자주하는 실수 및 해결책

**실수 1: 추상적이고 감정적인 표현**
```
❌ "열정을 가지고 있습니다"
✅ "관련 분야 5년 경력, 특허 2건 보유"

❌ "혁신적인 제품입니다"
✅ "기존 제품 대비 비용 30% 절감, 효율 2배 향상"

❌ "최고의 품질을 자랑합니다"
✅ "피부과 임상 테스트 완료, 보습 지속력 2배 입증"
```

**실수 2: 근거 없는 수치**
```
❌ "시장 규모가 큽니다"
✅ "국내 시장 규모 7조원 (한국무역협회, 2024)"

❌ "많은 고객이 원합니다"
✅ "100명 인터뷰 결과 78%가 구매 의향 표시"

❌ "빠르게 성장할 것입니다"
✅ "1년차 월 매출 500만원 → 2년차 3,000만원 목표"
```

**실수 3: 경쟁사 분석 부족**
```
❌ "경쟁사가 없습니다"
✅ "기존 제품 대비 가격 20% 낮고, 기능 3가지 추가"

❌ "우리가 최고입니다"
✅ [비교표로 객관적 차별점 제시]

❌ "차별화된 서비스입니다"
✅ "경쟁사 A는 중개만, 우리는 실시간 매칭 + 구독 서비스"
```

**실수 4: 비현실적인 목표**
```
❌ "1년 내 매출 100억"
✅ "1년차 5천만원 → 2년차 3억 → 3년차 10억"

❌ "글로벌 1위 달성"
✅ "국내 시장 점유율 5% 목표"

❌ "모든 고객층 공략"
✅ "초기 2030 여성 집중 → 단계적 확대"
```

**실수 5: 팀 구성 불명확**
```
❌ "좋은 팀원들이 있습니다"
✅ "CTO: 공대 석사, 기술 개발 7년 /
    CMO: 마케팅 팀장 5년"

❌ "혼자 다 합니다"
✅ "현재 대표+CTO, 하반기 마케터 1인 채용 예정,
    외부 세무법인 협력"

❌ "열심히 하겠습니다"
✅ "지난 3개월간 시제품 3회 개선, 체험단 피드백 100% 반영"
```

### 8.4 최종 제출 전 점검

**문서 품질:**
```
□ 맞춤법 검사 완료
□ 띄어쓰기 확인
□ 문장 호응 체크 ("~을 통해 ~한다" 등)
□ 숫자 단위 통일 (만원/억원)
□ 폰트 통일 (본문 9~11PT)
□ 줄 간격 통일 (140~180%)
□ 색상 일관성 (강조색 2~3개 이내)
```

**내용 완성도:**
```
□ PSST 구조 완성
□ 요약문 1~2페이지
□ 본문 15~20페이지
□ 이미지/표/그래프 10개 이상
□ 모든 주장에 근거 있음

□ 중복·모순 내용 없음
```

**제출 서류:**
```
□ 사업계획서 PDF (파일명 규칙 준수)

□ 요구 서류 (신분증 사본, 학력 증명 등)
□ 제출 기한 확인 (마감 최소 1일 전 제출 권장)
□ 제출 경로 확인 (온라인 접수 시스템 테스트)
```

6. 개발 진행도

7. 지원금 집행 계획

8. 기타 자금 조달 계획

9. 비즈니스 모델

10. 단계별 진입 전략

11. 중장기 로드맵

12. 사회적 가치 도입

13. 조직 구성
```

### 9.2 프롬프트 활용 방법

**기본 사용법:**
```
1. 각 프롬프트의 [여기에 입력] 부분에 구체적 정보 입력
2. ChatGPT에 복사하여 실행
3. 결과물을 검토하고 수정·보완
4. 자신의 언어로 재작성 (그대로 복사 금지)
```

**효과적인 활용 팁:**
```
✅ 한 번에 하나씩: 각 파트별로 순차적으로 작성
✅ 구체적 입력: "사업 아이템" 란에 최대한 자세히 입력
✅ 반복 개선: 첫 결과가 마음에 안 들면 정보를 추가해 재실행
✅ 비판적 검토: AI가 생성한 내용의 팩트 체크 필수
✅ 개인화: 자신만의 경험과 데이터로 보강
```

## 10. 핵심 요약

### 10.1 PSST 한눈에 보기

```
P - PROBLEM (문제 정의)
  │ ✓ 고객 세그먼트 명시
  │ ✓ PAIN POINT 수치화
  │ ✓ 경제적 손실 계산
  │ ✓ 최신 통계 (3년 이내)
  │ ✓ 사회적 맥락 연결
  │
S - SOLUTION (해결방안)
  │ ✓ 한 줄 정의
  │ ✓ 형태·기능·기술·디자인
  │ ✓ TAM·SAM·SOM
  │ ✓ 경쟁사 비교표
  │ ✓ 자사 강점 3가지
  │ ✓ 시장 진입 4단계
  │ ✓ 개발 진행 단계
  │ ✓ 실행 경험 증명
  │
S - SCALE-UP (성장전략)
  │ ✓ 비즈니스 모델 9요소
  │ ✓ 3단계 진입 전략
  │ ✓ 중장기 로드맵
  │ ✓ ESG 가치
  │ ✓ KPI 지표
  │
T - TEAM (조직구성)
    ✓ 대표자 역량 4요소
    ✓ 핵심 인력 구성
    ✓ 외부 네트워크
    ✓ 분야별 보완
    ✓ 채용 계획
```

### 10.2 합격의 3원칙

```
1. 논리적 구조
   → PSST가 자연스럽게 연결되는가?
   → 읽는 사람이 "아, 이해된다" 느끼는가?

2. 구체적 근거
   → 모든 주장에 수치와 데이터가 있는가?
   → 출처가 명시되어 있는가?

3. 실행 가능성
   → 실제로 해봤거나 할 수 있다는 증거가 있는가?
   → 팀 구성과 자금 계획이 현실적인가?
```

### 10.3 가장 중요한 한 문장

**정부가 보는 본질:**
> "나는 [누구]의 [문제]를 [무엇]으로 해결해 [어떻게] 돈을 벌겠다"

**이 한 문장이 명확하면 합격**

### 10.4 심사위원이 기억하는 3가지

```
1. 첫 1분 (요약문)
   → 핵심이 즉시 파악되는가?

2. 숫자 (시장 규모, 차별성, 매출)
   → 구체적이고 근거 있는 수치인가?

3. 태도 (Q&A 대응)
   → 자신감 있고 논리적으로 답변하는가?
```

### 10.5 절대 잊지 말 것

```
감정 < 논리
말 < 수치
계획 < 실행 경험
혼자 < 팀
당장 < 지속가능성
```
